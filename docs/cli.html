# CLI Usage ## End-to-end scoring The single entry point. Point it at a resume PDF. ```bash python score.py /path/to/resume.pdf ``` ### What happens 1. **PDF extraction** — converted to Markdown, cached to `cache/resumecache_.json` (when development mode is on). 2. **GitHub enrichment** — if a GitHub profile is found in the resume, repositories are fetched and cached to `cache/githubcache_.json`. 3. **Evaluation** — the evaluator scores the candidate and prints a report. In development mode, a CSV row is appended to `resume_evaluations.csv`. ### Example output ``` ── Evaluation Report ────────────────────────────── open_source ████████░░ 8 / 10 self_projects ██████░░░░ 6 / 10 production ██████████ 10 / 10 technical_skills ███████░░░ 7 / 10 bonus +2 deductions -1 ───────────────────────────── total 32 / 40 ``` ## Development mode Toggle caching and CSV export in `config.py`: ```python DEVELOPMENT_MODE = True # enables cache + CSV ``` Leave it on during iteration. Results accumulate in `resume_evaluations.csv` for batch analysis. ## Caching Intermediate files are stored under `cache/`: - `cache/resumecache_.json` — parsed resume data - `cache/githubcache_.json` — fetched GitHub data Clear the cache directory to force re-extraction.
